One Hour A Day

One Hour A Day is a social platform that aims to diversify contributors to development projects. Project owners post various products and tasks that need to be completed and volunteers are able to browse and complete these tasks based on their skill level. The tasks are supposed to be small easily managed chunks that can be completed in a few hours. My capacity on this project is UX Designer and Front-End Development. Out of the two functions I primarly focus on UX design and wireframe work for the various versions of the platform. Currently we are working on web, android, and iOS versions with an expected release date of late 2018.

First image depicts the initial top landing screen and the second image displays the middle section of the same page.

I have thoroughly enjoyed my time on this project as it has taught me many aspects of working within a team to accomplish a design goal. I have been able to learn about various programs and collaborating platforms that help streamline both my own workflow as well as the teams. In this project I have the responsibility of being the lead designer for the group. This function has taught me to really really work on time management and communication. In order to keep things running smoothly, I have develop a system to allow contributors to claim their own tasks while simoultaneously requesting various designers to attack certain problems. We have been pretty successful so far and can easily track both the progress of the project and the commitment level of our contributing designers. This has also been a unique experience for me as I have taken on mostly a design role instead of development which has allowed me to talk with our developers about the designs and how I can improve my wireframes to allow developers to understand them better.
